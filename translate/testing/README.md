# 测试部门 (Testing)

测试部门代理专注于质量保证、性能测试、API验证和基于证据的评估。

## 代理列表

| 文件名 | 代理名称 | 主要职责 |
|--------|----------|----------|
| testing-workflow-optimizer.md | 工作流优化者 | 流程分析、效率提升、持续改进 |
| testing-api-tester.md | API测试者 | API验证、集成测试、安全测试 |
| testing-evidence-collector.md | 证据收集者 | 基于截图的QA、视觉验证 |
| testing-test-results-analyzer.md | 测试结果分析者 | 测试评估、质量指标、数据分析 |
| testing-reality-checker.md | 现实检查者 | 基于证据的认证、怀疑评估、最终集成测试 |
| testing-tool-evaluator.md | 工具评估者 | 技术评估、工具选择 |
| testing-performance-benchmarker.md | 性能基准测试者 | 负载测试、性能优化、压力测试 |

## 测试覆盖范围

| 测试类型 | 目标 |
|----------|------|
| 功能测试 | 95%+ 端点覆盖率 |
| 安全测试 | 零关键漏洞 |
| 性能测试 | P95响应 < 200ms |
| 负载测试 | 10倍正常流量容量 |
| 视觉测试 | 截图对比验证 |

## 质量门

| 阶段 | 通过标准 |
|------|----------|
| 功能门 | 所有任务通过QA、无关键bug |
| 生产门 | 用户旅程完整、跨设备一致 |
| 发布门 | 部署成功、系统稳定 |

## 成功指标

- 任务首次QA通过 70%+
- API测试自动化 90%
- 性能SLA达成率 99%+
