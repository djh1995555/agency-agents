# ⚡ NEXUS快速启动指南

> **5分钟内从零到编排多代理管道。**

---

## 什么是NEXUS？

**NEXUS**（专家网络，策略统一）将The Agency的AI专家转变为协调的管道。不再是逐个激活代理并希望它们协同工作，NEXUS精确定义谁做什么、何时做以及如何在每一步验证质量。

## 选择你的模式

| 我想... | 使用 | 代理 | 时间 |
|---------|------|------|------|
| 从零构建完整产品 | **NEXUS-Full** | 全部 | 12-24周 |
| 构建功能或MVP | **NEXUS-Sprint** | 15-25 | 2-6周 |
| 做特定任务（bug修复、活动、审计） | **NEXUS-Micro** | 5-10 | 1-5天 |

## 🚀 NEXUS-Full：启动完整项目

**复制此提示词激活完整管道：**

```
以NEXUS-Full模式激活代理编排者。

项目：[你的项目名称]
规范：[描述你的项目或链接到规范]

执行完整的NEXUS管道：
- 阶段0：发现（趋势研究员、反馈综合者、UX研究员、分析报告员、法律合规检查者、工具评估者）
- 阶段1：策略（工作室制作人、高级项目经理、冲刺优先者、UX架构师、品牌守护者、后端架构师、财务追踪者）
- 阶段2：基础（DevOps自动化者、前端开发者、后端架构师、UX架构师、基础设施维护者）
- 阶段3：构建（开发↔QA循环 — 所有工程 + 证据收集者）
- 阶段4：加固（现实检查者、性能基准测试者、API测试者、法律合规检查者）
- 阶段5：发布（增长黑客、内容创作者、所有营销代理、DevOps自动化者）
- 阶段6：运营（分析报告员、基础设施维护者、支持响应者、持续进行）

每个阶段之间有质量门。所有评估需要证据。
每个任务最多3次重试后升级。
```

## 🏃 NEXUS-Sprint：构建功能或MVP

**复制此提示词：**

```
以NEXUS-Sprint模式激活代理编排者。

功能/MVP：[描述你正在构建的内容]
时间线：[目标周数]
跳过阶段0（市场已验证）。

冲刺团队：
- PM：高级项目经理、冲刺优先者
- 设计：UX架构师、品牌守护者
- 工程：前端开发者、后端架构师、DevOps自动化者
- QA：证据收集者、现实检查者、API测试者
- 支持：分析报告员

从阶段1开始进行架构和冲刺规划。
对所有实现任务运行开发↔QA循环。
发布前需要现实检查者批准。
```

## 🎯 NEXUS-Micro：做特定任务

**选择你的场景并复制提示词：**

### 修复Bug
```
激活后端架构者调查并修复[Bug描述]。
修复后，激活API测试者验证修复。
然后激活证据收集者确认无视觉回归。
```

### 运行营销活动
```
激活社交媒体策略师作为[活动描述]的活动负责人。
团队：内容创作者、Twitter互动者、Instagram策展人、Reddit社区构建者。
品牌守护者在发布前审查所有内容。
分析报告员每天追踪表现。
增长黑客每周优化渠道。
```

### 进行合规审计
```
激活法律合规检查者进行全面合规审计。
范围：[GDPR / CCPA / HIPAA / 全部]
审计后，激活执行摘要生成器创建利益相关者报告。
```

### 调查性能问题
```
激活性能基准测试者诊断性能问题。
范围：[API响应时间 / 页面加载 / 数据库查询 / 全部]
诊断后，激活基础设施维护者进行优化。
DevOps自动化者部署任何基础设施变更。
```

### 市场研究
```
激活趋势研究员进行[领域]的市场情报。
交付物：竞争格局、市场规模、趋势预测。
研究后，激活执行摘要生成器创建执行简报。
```

### UX改进
```
激活UX研究员识别[功能/产品]中的可用性问题。
研究后，激活UX架构师设计改进。
前端开发者实现变更。
证据收集者验证改进。
```

## 🔑 30秒关键概念

1. **质量门** — 无基于证据的批准不得进入下一阶段
2. **开发↔QA循环** — 每个任务先构建后测试；通过继续，失败重试（最多3次）
3. **交接** — 代理之间结构化的上下文转移（永不从零开始）
4. **现实检查者** — 最终质量权威；默认"需要工作"
5. **代理编排者** — 管理整个流程的管道控制器
6. **证据优于声明** — 截图、测试结果和数据——而非断言
