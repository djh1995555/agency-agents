# 🏗️ 第1阶段剧本 — 战略与架构

> **持续时间**：5-10天 | **代理**：8个 | **门控者**：工作室制作人 + 现实检查器

---

## 目标

在编写任何代码之前，定义我们要构建什么、如何构建以及成功标准。每个架构决策都有文档记录。每个功能都有优先级。每一分钱都有预算。

## 前置条件

- [ ] 第0阶段质量门已通过（推进决策）
- [ ] 第0阶段交接包已接收
- [ ] 利益相关者对项目范围达成一致

## 代理激活序列

### 步骤1：战略框架（第1-3天，并行）

#### 🎬 工作室制作人 — 战略组合对齐
```
激活工作室制作人进行 [项目] 的战略组合对齐。

输入：第0阶段执行摘要 + 市场分析报告
所需交付物：
1. 带项目定位的战略组合计划
2. 愿景、目标和ROI目标
3. 资源分配策略
4. 风险/回报评估
5. 成功标准和里程碑定义

对齐：组织战略目标
格式：战略组合计划模板
时间表：3天
```

#### 🎭 品牌守护者 — 品牌标识系统
```
激活品牌守护者进行 [项目] 的品牌标识开发。

输入：第0阶段UX研究（人物画像、旅程地图）
所需交付物：
1. 品牌基础（宗旨、愿景、使命、价值观、个性）
2. 视觉标识系统（颜色、排版、间距作为CSS变量）
3. 品牌声音和消息架构
4. 标志系统规范（如果是新品牌）
5. 品牌使用指南

格式：品牌标识系统文档
时间表：3天
```

#### 💰 财务追踪器 — 预算和资源规划
```
激活财务追踪器进行 [项目] 的财务规划。

输入：工作室制作人战略计划 + 第0阶段技术栈评估
所需交付物：
1. 带类别细分的全面项目预算
2. 资源成本预测（代理、基础设施、工具）
3. ROI模型及盈亏平衡分析
4. 现金流时间表
5. 财务风险评估及应急储备

格式：带ROI预测的财务计划
时间表：2天
```

### 步骤2：技术架构（第3-7天，并行，步骤1输出可用后）

#### 🏛️ UX架构师 — 技术架构 + UX基础
```
激活UX架构师进行 [项目] 的技术架构工作。

输入：品牌守护者视觉标识 + 第0阶段UX研究
所需交付物：
1. CSS设计系统（变量、令牌、比例）
2. 布局框架（网格/Flexbox模式、响应式断点）
3. 组件架构（命名约定、层次结构）
4. 信息架构（页面流程、内容层次）
5. 主题系统（浅色/深色/系统切换）
6. 无障碍基础（WCAG 2.1 AA基线）

要创建的文件：
- css/design-system.css
- css/layout.css
- css/components.css
- docs/ux-architecture.md

格式：开发者就绪基础包
时间表：4天
```

#### 🏗️ 后端架构师 — 系统架构
```
激活后端架构师进行 [项目] 的系统架构工作。

输入：第0阶段技术栈评估 + 合规要求
所需交付物：
1. 系统架构规范
   - 架构模式（微服务/单体/无服务器/混合）
   - 通信模式（REST/GraphQL/gRPC/事件驱动）
   - 数据模式（CQRS/事件溯源/CRUD）
2. 数据库架构设计及索引策略
3. API设计规范及版本控制
4. 认证和授权架构
5. 安全架构（纵深防御）
6. 可扩展性计划（水平扩展策略）

格式：系统架构规范
时间表：4天
```

#### 🤖 AI工程师 — ML架构（如适用）
```
激活AI工程师进行 [项目] 的ML系统架构工作。

输入：后端架构师系统架构 + 第0阶段数据审计
所需交付物：
1. ML系统设计
   - 模型选择和训练策略
   - 数据管道架构
   - 推理策略（实时/批量/边缘）
2. AI伦理和安全框架
3. 模型监控和重训练计划
4. 与主应用的集成点
5. ML基础设施成本预测

条件：仅在项目包含AI/ML功能时激活
格式：ML系统设计文档
时间表：3天
```

#### 👔 高级项目经理 — 规范到任务转换
```
激活高级项目经理进行 [项目] 的任务列表创建。

输入：所有第0阶段文档 + 架构规范（可用时）
所需交付物：
1. 全面任务列表
   - 引用规范中的确切需求（无豪华功能）
   - 每个任务有明确的验收标准
   - 任务间依赖关系映射
   - 工作量估算（故事点或小时）
2. 工作分解结构
3. 关键路径识别
4. 实施风险登记册

规则：
- 不要添加规范中没有的功能
- 引用需求的确切文本
- 工作量估算要现实

格式：带验收标准的任务列表
时间表：3天
```

### 步骤3：优先级排序（第7-10天，顺序执行，步骤2后）

#### 🎯 冲刺优先排序器 — 功能优先级排序
```
激活冲刺优先排序器进行 [项目] 的积压优先级排序。

输入：
- 高级项目经理 → 任务列表
- 后端架构师 → 系统架构
- UX架构师 → UX架构
- 财务追踪器 → 预算框架
- 工作室制作人 → 战略计划

所需交付物：
1. RICE评分积压（触达、影响、信心、工作量）
2. 基于速度估算的冲刺分配
3. 带关键路径的依赖图
4. MoSCoW分类（必须有/应该有/可以有/不会有）
5. 带里程碑映射的发布计划

验证：工作室制作人确认战略对齐
格式：优先级冲刺计划
时间表：2天
```

## 质量门清单

| # | 标准 | 证据来源 | 状态 |
|---|------|----------|------|
| 1 | 架构覆盖100%规范需求 | 高级项目经理任务列表与架构交叉引用 | ☐ |
| 2 | 品牌系统完整（标志、颜色、排版、声音） | 品牌守护者交付物 | ☐ |
| 3 | 所有技术组件有实施路径 | 后端架构师 + UX架构师规范 | ☐ |
| 4 | 预算已批准且在约束内 | 财务追踪器计划 | ☐ |
| 5 | 冲刺计划基于速度且现实 | 冲刺优先排序器积压 | ☐ |
| 6 | 安全架构已定义 | 后端架构师安全规范 | ☐ |
| 7 | 合规要求已整合到架构 | 法律要求映射到技术决策 | ☐ |

## 门决策

**需要双重签署**：工作室制作人（战略）+ 现实检查器（技术）

- **批准**：带完整架构包进入第2阶段
- **修订**：特定项目需要返工（返回相关步骤）
- **重构**：根本性架构问题（重启第1阶段）
