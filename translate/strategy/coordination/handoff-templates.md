# 📋 NEXUS 交接模板

> NEXUS 管道中每种代理间交接的标准化模板。一致的交接防止上下文丢失 — 多代理协调失败的第一大原因。

---

## 1. 标准交接模板

用于任何代理间工作转移。

```markdown
# NEXUS 交接文档

## 元数据
| 字段 | 值 |
|------|-----|
| **来自** | [代理名称]（[部门]） |
| **致** | [代理名称]（[部门]） |
| **阶段** | 第 [N] 阶段 — [阶段名称] |
| **任务参考** | [冲刺优先排序器积压中的任务ID] |
| **优先级** | [关键 / 高 / 中 / 低] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 上下文
**项目**：[项目名称]
**当前状态**：[到目前为止已完成的内容 — 要具体]
**相关文件**：
- [文件/路径/1] — [包含什么内容]
- [文件/路径/2] — [包含什么内容]
**依赖项**：[此工作依赖什么完成]
**约束**：[技术、时间表或资源约束]

## 交付物请求
**需要什么**：[具体、可衡量的交付物描述]
**验收标准**：
- [ ] [标准1 — 可衡量]
- [ ] [标准2 — 可衡量]
- [ ] [标准3 — 可衡量]
**参考材料**：[规范、设计、先前工作的链接]

## 质量期望
**必须通过**：[此交付物的具体质量标准]
**所需证据**：[完成证明的样子]
**交接给**：[谁接收输出及他们需要的格式]
```

## 2. QA反馈循环 — 通过

当证据收集器或其他QA代理批准任务时使用。

```markdown
# NEXUS QA裁决：通过 ✅

## 任务
| 字段 | 值 |
|------|-----|
| **任务ID** | [ID] |
| **任务描述** | [描述] |
| **开发者代理** | [代理名称] |
| **QA代理** | [代理名称] |
| **尝试** | [N] / 3 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决：通过

## 证据
**截图**：
- 桌面（1920x1080）：[文件名/路径]
- 平板（768x1024）：[文件名/路径]
- 移动（375x667）：[文件名/路径]

**功能验证**：
- [x] [验收标准1] — 已验证
- [x] [验收标准2] — 已验证
- [x] [验收标准3] — 已验证

**品牌一致性**：已验证 — 颜色、排版、间距匹配设计系统
**无障碍**：已验证 — 键盘导航、对比度、语义化HTML
**性能**：[测量的加载时间] — 在可接受范围内

## 备注
[任何观察、未来改进的次要建议或正面表扬]

## 下一步行动
→ 代理编排器：标记任务完成，推进到积压中的下一个任务
```

## 3. QA反馈循环 — 失败

当证据收集器或其他QA代理拒绝任务时使用。

```markdown
# NEXUS QA裁决：失败 ❌

## 任务
| 字段 | 值 |
|------|-----|
| **任务ID** | [ID] |
| **任务描述** | [描述] |
| **开发者代理** | [代理名称] |
| **QA代理** | [代理名称] |
| **尝试** | [N] / 3 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决：失败

## 发现的问题

### 问题1：[类别] — [严重程度：关键/高/中/低]
**描述**：[问题的确切描述]
**预期**：[根据验收标准应该发生什么]
**实际**：[实际发生了什么]
**证据**：[截图文件名或测试输出]
**修复说明**：[具体、可操作的解决说明]
**需修改的文件**：[确切的文件路径]

### 问题2：[类别] — [严重程度]
**描述**：[...]
**预期**：[...]
**实际**：[...]
**证据**：[...]
**修复说明**：[...]
**需修改的文件**：[...]

[继续列出所有发现的问题]

## 验收标准状态
- [x] [标准1] — 通过
- [ ] [标准2] — 失败（见问题1）
- [ ] [标准3] — 失败（见问题2）

## 重试说明
**给开发者代理**：
1. 只修复上面列出的问题
2. 不要引入新功能或更改
3. 解决所有问题后重新提交QA
4. 这是 [N] / 最多3次尝试

**如果第3次尝试失败**：任务将升级给代理编排器
```

## 4. 升级报告

当任务超过3次重试尝试时使用。

```markdown
# NEXUS 升级报告 🚨

## 任务
| 字段 | 值 |
|------|-----|
| **任务ID** | [ID] |
| **任务描述** | [描述] |
| **开发者代理** | [代理名称] |
| **QA代理** | [代理名称] |
| **已耗尽尝试** | 3/3 |
| **升级给** | [代理编排器 / 工作室制作人] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 失败历史

### 尝试1
- **发现的问题**：[摘要]
- **应用的修复**：[开发者更改了什么]
- **结果**：失败 — [为什么仍然失败]

### 尝试2
- **发现的问题**：[摘要]
- **应用的修复**：[开发者更改了什么]
- **结果**：失败 — [为什么仍然失败]

### 尝试3
- **发现的问题**：[摘要]
- **应用的修复**：[开发者更改了什么]
- **结果**：失败 — [为什么仍然失败]

## 根本原因分析
**任务为何持续失败**：[潜在问题分析]
**系统性问题**：[这是一次性问题还是模式？]
**复杂度评估**：[任务范围是否合理？]

## 建议解决方案
- [ ] **重新分配**给不同的开发者代理（[推荐的代理]）
- [ ] **分解**为更小的子任务（[建议的分解]）
- [ ] **修订方法** — 需要架构/设计变更
- [ ] **接受**当前状态并记录限制
- [ ] **推迟**到未来冲刺

## 影响评估
**阻塞**：[其他哪些任务被此阻塞]
**时间表影响**：[这如何影响整体进度]
**质量影响**：[如果我们接受当前状态存在什么质量妥协]

## 需要决策
**决策者**：[代理编排器 / 工作室制作人]
**截止时间**：[何时需要决策以避免进一步延误]
```

## 5. 阶段门交接

在NEXUS阶段之间转换时使用。

```markdown
# NEXUS 阶段门交接

## 转换
| 字段 | 值 |
|------|-----|
| **从阶段** | 第 [N] 阶段 — [名称] |
| **到阶段** | 第 [N+1] 阶段 — [名称] |
| **门控者** | [代理名称] |
| **门结果** | [通过 / 失败] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 门标准结果
| # | 标准 | 阈值 | 结果 | 证据 |
|---|------|------|------|------|
| 1 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 2 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 3 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |

## 向前传递的文档
1. [文档名称] — [下一阶段的目的]
2. [文档名称] — [下一阶段的目的]
3. [文档名称] — [下一阶段的目的]

## 下一阶段的关键约束
- [来自本阶段发现的约束1]
- [来自本阶段发现的约束2]

## 下一阶段代理激活
| 代理 | 角色 | 优先级 |
|------|------|--------|
| [代理1] | [下一阶段的角色] | [立即 / 第2天 / 按需] |
| [代理2] | [下一阶段的角色] | [立即 / 第2天 / 按需] |

## 向前传递的风险
| 风险 | 严重程度 | 缓解措施 | 负责人 |
|------|----------|----------|--------|
| [风险] | [P0-P3] | [缓解计划] | [代理] |
```

## 6. 冲刺交接

```markdown
# NEXUS 冲刺交接

## 冲刺摘要
| 字段 | 值 |
|------|-----|
| **冲刺** | [编号] |
| **持续时间** | [开始日期] → [结束日期] |
| **冲刺目标** | [目标陈述] |
| **速度** | [计划] / [实际] 故事点 |

## 完成状态
| 任务ID | 描述 | 状态 | QA尝试 | 备注 |
|--------|------|------|--------|------|
| [ID] | [描述] | ✅ 完成 | [N] | [备注] |
| [ID] | [描述] | ✅ 完成 | [N] | [备注] |
| [ID] | [描述] | ⚠️ 结转 | [N] | [原因] |

## 质量指标
- **首次QA通过率**：[X]%
- **平均重试次数**：[N]
- **完成任务**：[X/Y]
- **交付故事点**：[N]

## 结转到下一冲刺
| 任务ID | 描述 | 原因 | 优先级 |
|--------|------|------|--------|
| [ID] | [描述] | [为何未完成] | [RICE评分] |

## 回顾洞察
**什么做得好**：[关键成功]
**什么需要改进**：[关键改进]
**行动项**：[下一冲刺的具体变更]

## 下一冲刺预览
**冲刺目标**：[建议目标]
**关键任务**：[最高优先级项目]
**依赖项**：[跨团队依赖]
```

## 7. 事件交接

在事件响应期间使用。

```markdown
# NEXUS 事件交接

## 事件
| 字段 | 值 |
|------|-----|
| **严重程度** | [P0 / P1 / P2 / P3] |
| **检测者** | [代理或系统] |
| **检测时间** | [时间戳] |
| **分配给** | [代理名称] |
| **状态** | [调查中 / 缓解中 / 已解决 / 事后分析] |

## 描述
**发生了什么**：[事件的清晰描述]
**影响**：[谁/什么受影响及严重程度]
**时间线**：
- [HH:MM] — [事件]
- [HH:MM] — [事件]
- [HH:MM] — [事件]

## 当前状态
**受影响系统**：[列表]
**可用变通方案**：[是/否 — 如果是则描述]
**预计解决时间**：[时间估计]

## 已采取的行动
1. [采取的行动及结果]
2. [采取的行动及结果]

## 交接上下文
**给下一响应者**：
- [已尝试的方法]
- [尚未尝试的方法]
- [疑似根本原因]
- [需要检查的相关日志/指标]

## 利益相关者沟通
**最后更新发送**：[时间戳]
**下次更新到期**：[时间戳]
**沟通渠道**：[更新发布位置]
```

## 使用指南

| 情况 | 使用的模板 |
|------|-----------|
| 向另一代理分配工作 | 标准交接（#1） |
| QA批准任务 | QA通过（#2） |
| QA拒绝任务 | QA失败（#3） |
| 任务超过3次重试 | 升级报告（#4） |
| 阶段之间移动 | 阶段门交接（#5） |
| 冲刺结束 | 冲刺交接（#6） |
| 系统事件 | 事件交接（#7） |
